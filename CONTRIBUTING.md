# Contributing to Serac

Thank you for your interest in contributing to Serac.

This document explains how to contribute to the project, including reporting bugs, proposing features, improving documentation, and submitting code changes.

## What You Can Contribute

You can help in many ways:

- Fix bugs
- Propose or implement new features
- Improve documentation
- Add or improve tests
- Suggest UI and UX improvements
- Refactor code for clarity or performance

Even small improvements, such as typo fixes or documentation updates, are welcome.

## Before You Begin

Before making changes, please:

- Read the [README.md](README.md) file for project information and setup instructions
- Check existing issues to see whether your idea or bug has already been reported
- Look at open pull requests to understand current work and expectations

If your change is substantial, consider opening an issue first so the idea can be discussed before you start working on it.

## Setup

1. Fork the repository.
2. Clone your fork locally:
```
    git clone https://github.com/<your-username>/Serac.git
    cd Serac
```
3. Install any dependencies listed in the [README](README.md).
4. Create a new branch for your work:
```
    git checkout -b feature/your-feature-name
```
Use a branch name that clearly describes your change.

## Making Changes

When contributing code, please:

- Follow the existing style and structure of the project
- Keep changes focused and easy to review
- Write clear, readable code
- Add comments where needed
- Include tests when possible

## Commit Messages

Write commit messages that clearly describe your changes. Examples:
```
    git commit -m "Fix timer reset behavior"
    git commit -m "Add documentation for setup steps"
    git commit -m "Improve button layout on mobile"
```
## Pull Request Process

When your changes are ready:

1. Push your branch to your fork.
2. Open a pull request against the main branch of the original repository.
3. Describe what you changed and why.
4. Link any related issues.
5. Respond to review comments and make updates if requested.

## Reporting Bugs

If you find a bug, please open an issue and include:

- A clear description of the problem
- Steps to reproduce it
- Expected and actual behavior
- Screenshots or logs, if helpful

## Code of Conduct

Please be respectful, constructive, and kind in all interactions.

## Thanks

Thank you for helping improve Serac.
